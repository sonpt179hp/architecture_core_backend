# README

Mô tả ngắn nhiệm vụ của từng file trong thư mục gốc:

- `database_architecture.md`: Mô tả định hướng kiến trúc cơ sở dữ liệu cho hệ thống quản lý văn bản, gồm multi-tenancy, ltree cho cây tổ chức và partitioning cho bảng lớn.
- `design_pattern_architecture.md`: Mô tả định hướng tổ chức code backend, gồm Clean Architecture, Pragmatic DDD, CQRS và giao tiếp event-driven.
- `distributed_system_design.md`: Phân tích quyết định chọn nền tảng triển khai Dapr, so sánh Docker Swarm với Kubernetes và đưa ra khuyến nghị áp dụng.
- `backend_core_technical_guidelines.md`: Bổ sung các technical guardrails cho backend production-ready như security, pipeline xử lý chung, resilience, observability, testing và EF Core vận hành.
- `technology_comparison.md`: So sánh hệ thống cũ với kiến trúc mới về runtime, database, code architecture, giao tiếp service và hạ tầng triển khai.
- `old_system.md`: Tóm tắt các hạn chế chính của hệ thống cũ làm cơ sở cho việc đề xuất kiến trúc mới.
